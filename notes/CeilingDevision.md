# 올림 나눗셈

"남는 게 있으면 무조건 하나 더 필요하다"는 모든 상황에 등장.
피자 문제, 페이지네이션, 버스 좌석 배치, 서버 개수 계산 등.

- 일반 나눗셈: 실수 결과를 줄 수 있음. (8/7=1.142 ...)
- 정수 나눗셈: 내림(버림) 해서 정수로 만듬. (python - 8//7=1)
- **올림 나눗셈: 항상 위로 올림 (ceil(8/7)=2)**

---

## 쉽게 이해하기

피자 한 판에 `slice`조각이야. `n`명한테 한 조각씩 나눠주려면 피자 몇 판 시켜야 할까?

- n=8, slice=7 이면 → 8 ÷ 7 = 1판 + 1명 남음
- 1판만 시키면 7명밖에 못 먹어. 1명이 남잖아.
- 피자 반 판은 팔지 않으니까 → 무조건 1판 더 → 총 2판

**"남는 게 1명이든 6명이든, 모자라면 무조건 하나 더."**

---

## 코드로 쓰는 방법 3가지

### 방법 1: if/else로 직접 분기

```python
if n % slice == 0:
    answer = n // slice      # 딱 나누어 떨어지면 그냥 몫
else:
    answer = n // slice + 1  # 남는 게 있으면 한 판 더
```

가장 직관적. 조건이 명확하게 보임.

### 방법 2: (n + slice - 1) // slice 트릭

```python
answer = (n + slice - 1) // slice
```

`slice - 1`을 더하는 이유: slice로 나눈 나머지의 최대값이 `slice - 1`이기 때문.
나머지가 1이든 `slice-1`이든, `+(slice-1)`을 더하면 무조건 다음 배수를 넘어감.

```
n=8, slice=7  → (8+6)//7  = 14//7 = 2  ✓
n=14, slice=7 → (14+6)//7 = 20//7 = 2  ✓
n=15, slice=7 → (15+6)//7 = 21//7 = 3  ✓
```

### 방법 3: (n - 1) // slice + 1 트릭

```python
answer = (n - 1) // slice + 1
```

n에서 1을 빼면 "딱 나누어 떨어지는 경우"도 한 칸 아래로 내려가서,
`//`로 내림한 뒤 `+1`을 해주면 항상 올림과 같은 결과가 나옴.

```
n=7, slice=7  → (7-1)//7 + 1 = 0+1 = 1  ✓  (딱 맞는 경우도 정확히 처리)
n=8, slice=7  → (8-1)//7 + 1 = 1+1 = 2  ✓
n=14, slice=7 → (14-1)//7 + 1 = 1+1 = 2 ✓
n=15, slice=7 → (15-1)//7 + 1 = 2+1 = 3 ✓
```

---

## 세 방법 비교

| 방법 | 코드 | 특징 |
|------|------|------|
| if/else | `if n%s==0: n//s else: n//s+1` | 읽기 쉬움, 길다 |
| n+s-1 트릭 | `(n+slice-1)//slice` | 한 줄, 이유 이해하면 자연스러움 |
| n-1 트릭 | `(n-1)//slice+1` | 한 줄, 가장 많이 쓰이는 관용구 |

세 방법 모두 항상 같은 결과를 냄.
