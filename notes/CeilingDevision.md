# 올림 나눗셈

"남는 게 있으면 무조건 하나 더 필요하다"는 모든 상황에 등장.
피자 문제, 페이지네이션, 버스 좌석 배치, 서버 개수 계산 등.

- 일반 나눗셈: 실수 결과를 줄 수 있음. (8/7=1.142 ...)
- 정수 나눗셈: 내림(버림) 해서 정수로 만듬. (python - 8//7=1)
- **올림 나눗셈: 항상 위로 올림 (ceil(8/7)=2)**

---

## 쉽게 이해하기

피자 한 판에 7조각이야. 8명한테 한 조각씩 나눠주려면 피자 몇 판 시켜야 할까?

- 8 ÷ 7 = 1판 + 1조각 남음
- 근데 1판만 시키면 7명밖에 못 먹어. 1명이 남잖아.
- 피자 반 판은 팔지 않으니까 → 무조건 1판 더 시켜야 해 → 총 2판

이게 올림 나눗셈이야. **"남는 게 1명이든 6명이든, 모자라면 무조건 하나 더."**

---

## 코드로 쓰는 방법 2가지

### 방법 1: if/else로 직접 분기

```python
if n % 7 == 0:
    answer = n // 7      # 딱 나누어 떨어지면 그냥 몫
else:
    answer = n // 7 + 1  # 남는 게 있으면 한 판 더
```

- `n // 7` → 지금 피자 몇 판인지 계산
- `+ 1` → 한 판 더 추가

### 방법 2: (n + 6) // 7 트릭

```python
answer = (n + 6) // 7
```

왜 6을 더하냐면, 7로 나눈 나머지는 최대 6이기 때문이야.
(나머지가 7이 되는 순간 몫이 1 올라가버리니까, 나머지는 0~6 사이만 가능해.)

- 나머지가 있을 때(1~6): +6 하면 다음 7의 배수를 넘어감 → //7 하면 1 올라감
- 나머지가 없을 때(0): +6 해도 다음 7의 배수에 못 미침 → //7 해도 그대로

```
n=8  → (8+6)//7  = 14//7 = 2  ✓
n=14 → (14+6)//7 = 20//7 = 2  ✓
n=15 → (15+6)//7 = 21//7 = 3  ✓
```

두 방법은 항상 같은 결과를 냄. 방법 2가 코드 한 줄로 끝나서 더 짧음.
