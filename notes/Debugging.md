# 디버깅

## 한 줄 설명

디버깅은 기대한 결과와 실제 결과가 달라지는 원인을 관찰하고 범위를 좁혀 찾아가는 과정이다.

## 언제 떠올리는가

- 실행 중 오류가 발생할 때
- 실행은 되지만 결과가 틀릴 때
- 조건문이나 반복문이 실행되지 않는 것처럼 보일 때
- 변수의 값이 예상과 다를 때

## 내가 헷갈린 부분

디버깅용 `print()`는 문제를 직접 고치는 명령이 아니다. 현재 변수와 조건의 값을 보여 주어 문제가 시작되는 위치를 찾게 하는 도구다.

## 기본 순서

1. 실패한 입력으로 다시 실행한다.
2. 반복 입력이 번거로우면 해당 값을 잠시 하드코딩한다.
3. 의심되는 코드의 실행 전후 값을 출력한다.
4. 조건식 자체의 `True` 또는 `False`를 출력한다.
5. 어디까지 정상인지 확인하며 범위를 좁힌다.
6. 수정 후 다시 실행한다.
7. 디버깅용 코드와 하드코딩을 제거한다.

## 최소 예제

```python
day = "Sunday"
daily_wages = 120

print("condition:", day == "sunday")

if day == "sunday":
    print("before:", daily_wages)
    daily_wages * 2
    print("after:", daily_wages)
```

여기서 `"Sunday" == "sunday"`는 대소문자가 달라 `False`이고, `daily_wages * 2`는 계산 결과를 저장하지 않는다.

```python
if day == "Sunday":
    daily_wages *= 2
```
